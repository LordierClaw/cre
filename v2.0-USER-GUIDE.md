# CRE v2.0 User Guide

Welcome to CRE (Context Reconstruction Engine) v2.0. This version introduces significant refactorings, optimized context output, and new exploration tools.

## Key Changes in v2.0

- **Removal of NodeIds**: We have completely moved away from complex string identifiers (`fqn::signature::origin`). You now interact with the engine using human-readable **Symbols** (e.g., `UserController`, `UserService.save`).
- **Spring DI Architecture**: The core engine is now fully integrated with Spring Dependency Injection for better maintainability and performance.
- **Token Optimization**: Context reconstruction now uses structured XML-like tags and aggressive pruning of irrelevant code logic and comments to save tokens.

## Exploration Tools

Before requesting full context, use these tools to understand the project structure.

### `get_project_structure`
Returns a tree-like view of the project's directory and package structure.
- **Input**: `{"project_root": "/path/to/project"}`
- **Output**: A formatted text tree.

### `get_file_structure`
Returns a "skeleton" of a Java file (classes, properties, method signatures) without method bodies.
- **Input**: `{"project_root": "/path/to/project", "symbol": "UserController"}`
- **Output**: Code-like skeleton.

## Context Reconstruction (`get_context`)

The primary tool for getting code context. It now supports granular control via an `options` JSON object.

### Options Schema
```json
{
  "definitions": {
    "imports": "omitted|relevance|full",
    "properties": "omitted|relevance|full",
    "functions": "omitted|full"
  },
  "expanded_functions": ["com.pkg.Class.method1"]
}
```

- **Default Settings**: `imports: omitted`, `properties: relevance`, `functions: omitted`.
- **Relevance Mode**: Only includes imports and properties used by the methods currently being shown.
- **`expanded_functions`**: A list of methods (using Full FQN.Method format) that should be shown in **full**, overriding global settings.

### Transitive Expansion
The `depth` parameter (integer) controls how deep to traverse dependencies. Each class encountered is wrapped in `<ClassName>...</ClassName>` tags and returned in distance-based order.

## Error Handling

Standardized error messages are now provided for common failure scenarios:
- `[ProjectNotFoundException]`: The requested project root is missing or inaccessible.
- `[SymbolNotFoundException]`: The provided class or method symbol could not be resolved.
- `[IndexingException]`: A failure occurred during project ingestion.

## Example Request
```json
{
  "project_root": "/home/user/my-project",
  "node_id": "UserController.create",
  "depth": 1,
  "options": {
    "definitions": {
      "imports": "relevance"
    },
    "expanded_functions": ["com.myapp.UserController.create"]
  }
}
```
