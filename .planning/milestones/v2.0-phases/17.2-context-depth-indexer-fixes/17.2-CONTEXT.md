# Phase 17.2: Context Depth & Indexer Fixes

## Objective
Fix `get_context` depth logic to be more precise (Strict Reachability) and functional for `depth > 2`.

## Decisions
1.  **Depth Logic**:
    -   **`depth = 0`**: Return only the target function code, wrapped in its `<file>` tag. 
    -   **`depth > 0`**: Include the target function AND the specific callees reached by the traversal (at the specified depth).
    -   **Strict Reachability**: Omit all functions and properties that are NOT part of the reachability set for the given depth.
2.  **Edge Filtering**:
    -   `gatherNodesOrdered` must be selective. For context reconstruction, it should primarily follow `CALLS`, `USES_FIELD`, and `IMPLEMENTS`.
    -   Avoid jumping from Method to Class via `BELONGS_TO` and then to all other methods via `ENTRY_POINT`/`SERVICE_LAYER`.
3.  **Indexer Improvement**:
    -   Enhance `JavaAstIndexer.resolveCallee` to handle more complex signatures, reducing `?` fallback in argument types.
