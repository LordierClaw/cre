# Plan: Phase 17.1 - Output Format Migration

<objective>
Standardize the XML output format to use `<file name="...">` tags, improving token efficiency and consistency for LLM consumption.
</objective>

<tasks>

<task id="1" type="auto">
  <name>Task 1: Update CreServiceImpl output logic</name>
  <files>src/main/java/com/cre/core/service/CreServiceImpl.java</files>
  <action>
    Modify `buildIntegratedView` to:
    1. Extract the fully qualified name (FQN) of the type.
    2. Wrap the generated code in `<file name="FQN">` tags.
    3. Remove the legacy logic that replaced type code with `<ClassName>` tags.
    4. Ensure `<omitted_functions/>` and `<omitted_properties/>` tags are still correctly placed within the new `<file>` tag.
  </action>
  <verify>
    <automated>mvn test -Dtest=RealProjectE2ETest</automated>
  </verify>
  <done>true</done>
</task>

<task id="2" type="auto">
  <name>Task 2: Create Python verification script</name>
  <files>scripts/test_context_format.py</files>
  <action>
    Create a script that:
    1. Invokes the CRE MCP tool for a known symbol (e.g., `AdminBookController::createBook`).
    2. Asserts that the response contains `<file name="com.bookstore.controller.AdminBookController">`.
    3. Asserts that the response does NOT contain `<AdminBookController>`.
  </action>
  <verify>
    <automated>python3 scripts/test_context_format.py</automated>
  </verify>
  <done>true</done>
</task>

</tasks>
