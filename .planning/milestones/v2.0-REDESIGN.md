# Milestone v2.0: Redesign & Efficiency

## Objective
Refactor the core engine for better maintainability, simplify the toolset, and optimize context reconstruction for token efficiency.

## Requirements

### R-REF-01: Core Refactoring & Simplification
- [x] **REF-01.1**: Remove heuristic ranking and pruning logic (moving toward deterministic, on-demand expansion).
- [x] **REF-01.2**: Update `ProjectManager` to use Spring Dependency Injection.
- [x] **REF-01.3**: Centralize all tool/API functionality into service classes with clear interfaces/implementations.
- [x] **REF-01.4**: Remove legacy tools: `TraceFlowTool`, `FindImplementationsTool`, etc.

### R-CORE-02: Enhanced Project Exploration
- [x] **CORE-02.1**: Implement `index_project` (triggered automatically on-demand).
- [x] **CORE-02.2**: Implement `get_project_structure` with options for folder/package listing.
- [x] **CORE-02.3**: Implement `get_file_structure` (classes, properties, functions) with options for params and annotations.

### R-CTX-03: Optimized Context Reconstruction
- [x] **CTX-03.1**: Redesign `get_context` output with XML-like tags for classes.
- [x] **CTX-03.2**: Support depth-based expansion for transitive dependencies.
- [x] **CTX-03.3**: Support `options` JSON for granular control over imports, properties, and functions (`omitted|relevance|full`).
- [x] **CTX-03.4**: Handle circular dependencies and recursion in traversal.
- [x] **CTX-03.5**: Implement post-processing filter architecture for code logic omission.

### R-BUG-04: Robustness & Exception Handling
- [x] **BUG-04.1**: Skip unnecessary comments, but handle commented-out functions appropriately.
- [x] **BUG-04.2**: Standardize exception handling (File not found, null checks, Optional).

## Verification Criteria
- [ ] Unit tests for new structure tools.
- [ ] E2E tests for `get_context` with various depth and option configurations.
- [ ] Verification of reduced token usage via XML-like tagging and omission tags.
