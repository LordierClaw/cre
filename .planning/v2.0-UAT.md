# UAT: Milestone v2.0 Redesign & Efficiency

## Objective
Validate the new exploration tools, granular context options, and robust exception handling from a user's perspective.

## Test Sessions

| ID | Test Case | Status | Notes |
|----|-----------|--------|-------|
| T1 | `get_project_structure` returns tree view | Pass | Tree view correctly shows directories and files. |
| T2 | `get_file_structure` returns code skeleton | Pass | Returns class skeleton without method bodies. |
| T3 | `get_context` with default options (relevance) | Pass | Correctly includes used fields and prunes unused ones. |
| T4 | `get_context` with `expanded_functions` | Pass | Successfully overrides global 'omitted' for specific functions. |
| T5 | `get_context` with custom `omitted` levels | Pass | Content is correctly pruned when all definitions are set to 'omitted'. |
| T6 | Symbol resolution (Simple vs FQN) | Pass | Both simple names and FQNs are correctly resolved to the target node. |
| T7 | Standardized Error: Symbol Not Found | Pass | Returns `[SymbolNotFoundException]`. |
| T8 | Standardized Error: Project Not Found | Pass | Returns `[ProjectNotFoundException]`. |


---

## T1: `get_project_structure` returns tree view
**Goal**: Verify that the tool returns a human-readable tree of the project.

**Action**: Call `get_project_structure` for `cre` project.
**Expectation**: A string starting with "Project Structure:" followed by a `tree`-like output showing `src`, `pom.xml`, etc.

---

## T2: `get_file_structure` returns code skeleton
**Goal**: Verify that the tool returns a class skeleton without method bodies.

**Action**: Call `get_file_structure` for `CreServiceImpl`.
**Expectation**: Java source code showing class members and method signatures, but method bodies should be empty or abstract-like.

---

## T3: `get_context` with default options (relevance)
**Goal**: Verify that default relevance mode only includes used properties and imports.

**Action**: Call `get_context` for `CreController.getContext`.
**Expectation**: 
- Contains `CreService` field (it's used).
- Contains relevant imports.
- Wrapped in `<CreController>` tags.

---

## T4: `get_context` with `expanded_functions`
**Goal**: Verify specific functions can be expanded while others remain omitted.

**Action**: Call `get_context` for `CreController` with `expanded_functions: ["com.cre.mcp.CreController.getContext"]`.
**Expectation**: `getContext` body is full, other methods are replaced by `<omitted_functions/>`.

---

## T5: `get_context` with custom `omitted` levels
**Goal**: Verify `imports: omitted` and `properties: omitted` work.

**Action**: Call `get_context` with all definitions set to `omitted`.
**Expectation**: Contains `<omitted_imports/>` and `<omitted_properties/>` tags.

---

## T6: Symbol resolution (Simple vs FQN)
**Goal**: Verify symbol resolution works for both simple and full names.

**Action**: Call `get_context` with `CreServiceImpl` and then with `com.cre.core.service.CreServiceImpl`.
**Expectation**: Both return the same valid context.

---

## T7: Standardized Error: Symbol Not Found
**Goal**: Verify the error format for missing symbols.

**Action**: Call `get_context` with a non-existent class.
**Expectation**: Error message starting with `[SymbolNotFoundException]`.

---

## T8: Standardized Error: Project Not Found
**Goal**: Verify the error format for missing projects.

**Action**: Call `get_context` with a fake project root.
**Expectation**: Error message starting with `[ProjectNotFoundException]`.
